# Code Sync Bridge VSCode 扩展

Code Sync Bridge VSCode扩展为代码同步桥接服务提供图形化界面，让开发者能够在VSCode中方便地管理代码流同步。

## 功能特性

### 🌊 代码流管理
- **代码流浏览器**: 在侧边栏显示所有可用的代码流
- **一键拉取**: 直接从VSCode界面拉取代码流到本地
- **状态监控**: 实时显示代码流的同步状态和最后更新时间
- **详细信息**: 查看代码流的详细元数据和配置信息

### 📤 自动提交功能
- **文件监控**: 自动监听文件保存事件
- **智能标记**: 自动标记待提交的文件变更
- **一键提交**: 通过状态栏按钮快速提交所有变更
- **自动提交**: 可选的自动提交功能，保存后自动同步

### 📊 状态显示
- **状态栏指示器**: 显示当前同步状态和最后同步时间
- **进度提示**: 同步过程中显示详细进度信息
- **错误处理**: 友好的错误提示和重试机制

### ⚙️ 配置管理
- **SFTP配置**: 图形化配置SFTP连接参数
- **自动提交设置**: 启用/禁用自动提交功能
- **通知设置**: 控制各种通知的显示

## 安装和配置

### 1. 安装扩展

从VSCode扩展市场搜索"Code Sync Bridge"并安装，或者从本地安装：

```bash
cd packages/vscode-extension
npm run build
npm run package
code --install-extension code-sync-bridge-vscode-1.0.0.vsix
```

### 2. 配置SFTP连接

打开VSCode设置，搜索"Code Sync Bridge"，配置以下参数：

- **SFTP服务器地址**: 内外网桥接的SFTP服务器地址
- **SFTP端口**: SFTP服务器端口（默认22）
- **SFTP用户名**: 用于连接SFTP的用户名
- **自动提交**: 是否启用自动提交功能
- **监控间隔**: 代码流状态检查间隔（秒）

### 3. 验证配置

配置完成后，扩展会自动验证连接并显示可用的代码流。

## 使用指南

### 代码流管理

1. **查看代码流**: 在侧边栏的"代码流管理"面板中查看所有可用的代码流
2. **拉取代码流**: 点击代码流旁的下载图标，选择本地保存目录
3. **查看详情**: 展开代码流节点查看详细信息和状态
4. **刷新列表**: 点击刷新按钮获取最新的代码流列表

### 代码提交

1. **自动标记**: 保存文件后，扩展会自动标记为待提交状态
2. **查看待提交**: 状态栏会显示待提交文件数量
3. **手动提交**: 点击状态栏的"提交变更"按钮
4. **自动提交**: 启用后会在文件保存5秒后自动提交

### 管理面板

点击"打开管理面板"按钮可以打开详细的代码流管理界面，提供：

- 代码流卡片视图
- 详细状态信息
- 批量操作功能
- 可视化配置界面

## 命令面板

扩展提供以下命令（Ctrl+Shift+P打开命令面板）：

- `Code Sync Bridge: 刷新代码流` - 刷新代码流列表
- `Code Sync Bridge: 提交变更` - 提交待处理的变更
- `Code Sync Bridge: 打开设置` - 打开扩展设置
- `Code Sync Bridge: 打开管理面板` - 打开详细管理界面

## 状态指示器

状态栏显示当前同步状态：

- ✅ **就绪**: 系统正常，可以进行操作
- 🔄 **同步中**: 正在进行代码同步
- ⏳ **加载中**: 正在加载数据
- ❌ **错误**: 发生错误，点击查看详情
- 🔌 **离线**: 网络连接异常

## 故障排除

### 连接问题

如果无法连接到SFTP服务器：

1. 检查网络连接和防火墙设置
2. 验证SFTP服务器地址和端口
3. 确认用户名和动态码是否正确
4. 查看VSCode开发者控制台的错误信息

### 同步问题

如果代码同步失败：

1. 检查本地文件权限
2. 确认工作区目录结构正确
3. 查看是否有文件被其他程序占用
4. 重启VSCode并重新尝试

### 性能问题

如果扩展运行缓慢：

1. 增加监控间隔时间
2. 排除不必要的文件类型
3. 关闭自动提交功能
4. 检查本地磁盘空间
